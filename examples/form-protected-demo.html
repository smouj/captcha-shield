Client-Side Only (Demo/Example)

⚠️ Important Security Notice

This is a client-side demo only. CAPTCHA Shield runs entirely in the browser and provides NO production-grade security. For real security, you need server-side verification with signed tokens and nonce validation.

What This Demo Shows

This example demonstrates how CAPTCHA Shield can be embedded in a form. The widget collects behavioral data, generates challenges, and returns a result. However, all this happens in the browser without server verification, making it possible to bypass.

This is for demonstration and educational purposes only.

⚠️ Production Requirement

For production use: Never rely on client-side CAPTCHA alone. You must implement server-side verification with:

  • Cryptographically signed tokens
  • Nonce validation with TTL expiration
  • Rate limiting per IP/session
  • Server-side risk analysis
  • Audit logging for security events
